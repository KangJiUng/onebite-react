## 섹션 3: Node.js 기초

- React.js는 Node.js를 기반으로 동작하는 기술

  - 외에 Next.js, Vue.js, Svelte 등도 마찬가지’

- Node.js는 웹 브라우저가 아닌 환경에서도 자바스크립트 코드를 실행시켜주는 자바스크립트의 런타임, 즉 실행 환경(구동기)

  - 라이브러리나 언어가 아니라 컴퓨터에 직접 설치해서 사용하는 프로그램

- 자바스크립트는 태생이 오직 웹페이지 안에서 일어나는 아주 단순한 기능만을 개발하기 위해서 만들어진 간단한 스크립트 언어

  - 그렇기 때문에 동시대에 존재했던 C언어나 자바 언어와는 다르게 그 문법 자체가 매우 유연하고 또 작성하기가 편리하도록 설계가 되었다.
  - 사람들은 자바스크립트로 웹페이지 내부의 기능을 만드는 걸 넘어서 이제는 웹 브라우저 바깥에서도 자바스크립트를 이용해서 프로그램을 만들고 싶어했고 Node.js가 등장하게 되면서 JavaScript는 어디서든 동작할 수 있는 아주 범용적인 언어가 되었다.

- NPM이란, Node Package Manager라는 뜻으로 Node.js의 프로젝트 단위인 패키지를 관리하는 도구

  - 새로운 패키지 생성, 외부 라이브러리 설치 또는 삭제 등 유용한 기능들을 제공

- 프로젝트: 어떠한 목적을 갖는 프로그램의 단위(ex-쇼핑몰 프로젝트, 웹 포털 프로젝트 등)

  - Node.js에서는 이 프로젝트를 Package라고 부름
  - 모든 JavaScript 앱, 모든 React 앱들도 다 Package라는 단위로 만들어지게 되며, 모든 라이브러리들도 모두 다 이런 Package라는 단위로 만들어져 있다

- 모듈: 프로그램 내부를 기능별 단위로 분할한 부분
- 모듈 시스템: JS 코드를 기능별로 분리하여 여러 파일로 나누어 구성하고, 서로 가져와 쓰게 돕는 시스템

- 라이브러리: 프로그램을 개발할 때 필요한 다양한 기능들을 미리 만들어 모듈화 해 놓은 것

- package.json: 생성한 프로젝트의 메타정보와 프로젝트가 의존하고 있는 모듈들에 대한 정보들을 json 형태로 모아놓은 파일, 대략적인 의존성의 버전을 관리. npm istall 을 할 때 기준이 되어주는 파일
- package-lock.json: 프로젝트에 설치되어야하는 의존성의 정확한 버전을 관리해주는 파일. package.json는 버전의 대략적인 범위를 보여주는 반면 package-lock.json은 구체적인 버전명이 명시되어있다.
- node_modules 디렉토리: package.json에 있는 모듈과, 그 모듈들이 의존하고 있는 모든 모듈들을 포함
